# Project EDGE Design Philosophy

Every rule exists for one reason:

**To improve the experience at the table.**

If a mechanic slows the game without creating meaningful decisions, it should be simplified, redesigned, or removed.

---

# Core Principles

## Fast

Rules should be easy to learn and quick to reference.

Players should spend more time playing than searching through rulebooks.

---

## Lethal

Combat must remain dangerous.

Violence should always carry consequences.

---

## Cinematic

Rules should encourage memorable scenes instead of mechanical optimization.

Style matters.

---

## Accessible

New players should be able to create a character and understand the basics within a single session.

---

## Narrative First

Mechanics support the story.

They should never become obstacles to it.

---

# Design Goals

Project EDGE aims to:

* Reduce unnecessary complexity.
* Minimize bookkeeping.
* Keep players engaged.
* Preserve Cyberpunk RED's identity.
* Encourage creative problem solving.

---

# Design Process

Every proposed change should answer three questions:

1. Does this improve gameplay?
2. Does this reduce unnecessary complexity?
3. Does this preserve the intended experience?

If the answer is "no" to any of these questions, the change should be reconsidered.
